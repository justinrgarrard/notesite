Description

Notesite is a simple app for displaying Markdown notes stored in their own Git repository.

Reading Notes

Go to the notes page to navigate between topics and entries.

Writing Notes

Notes are stored in their own Git repository and brought in via a Git Submodule. Simply push your changes on the notes repository, pulling to this repository when you're ready to update the website.

Why?

I've tried a few other approaches to note-taking, but haven't been entirely satisfied with any of them.

  • Paper smudges and can't be edited easily.
  • Plain text files don't have markup tools.
  • Word documents aren't easy to organize across multiple topics.
  • LaTex and HTML files aren't very readable.
  • Wikis need their own infrastructure, like databases or hosting services.
  • Note-taking apps (like OneNote and Cherrytree) are difficult to share.

Using directories and Markdown seemed like a good barebones strategy. Authors get full control over how they structure their notebook. They also get markup tools (bold, italics, etc.) without affecting legibility of the raw file.

Jekyll adds readability, searchability, and ease of publication. Authors can choose to theme their notes if they want something a little fancier than raw text files.
